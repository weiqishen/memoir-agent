# 连载倾听草稿引擎 (Listener Workflow)

当用户输入 `/listen` 或者表明“我要一段段倒苦水”时，系统将进入“纯倾听者”的安全缓存模式。
此时，**任何情况下都绝对禁止真正走 `/recall` 的正式提取和 Python 写入流程。**

## 执行步骤：
1. **获取原文与图片**：读取用户此时扔进来的这句零散描述或者提供的截图内容。
2. **安静的书记员机制**：调用工具（比如先 `view_file` 看一下现在的草稿文件内容，如果内容不存在，则直接用 `write_to_file` 新建文件，如果存在，则使用原生文件操作工具如 `write_to_file` 或 `replace_file_content`，将原有的文本加上新到的一段话，合二为一重新安全写入。由于文件编辑工具的安全限制，**绝不允许使用含有 cat 或 echo >> 的 bash 脚本来追加内容**）。
   - 固定草稿路径：`memoirs/.draft_buffer.md`
   - 将这轮新的事实陈述直接作为普通文本放在文件结尾。
3. **极简回应**：给出简短、没有侵略性的共情倾听回应（不能长篇大论）。例如：“*记下来了，接着后来呢？*”，“*这种感觉一定很让人火大，继续吐苦水吧。*”
4. 等待用户的下一次投喂或直接下发 `/commit` 指令打包。
