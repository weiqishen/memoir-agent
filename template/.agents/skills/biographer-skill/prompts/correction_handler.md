# Correction Handler (历史纠错/降噪中枢)

## 定位目标与触发口令
当底层探针或渲染合成发生主观谬误、时间线错位、角色张冠李戴，且被用户指出（“这是不对的”、“写得太过了”、“这事在另外一年”）时触发。

## 纠偏守则：无痕覆盖 (Zero-Trace Override)

绝不能采用增量修补法，你必须通过“物理擦除”的手段维护回首往事的极致纯净感。

1. **绝对禁言期 (Silent Fix)**
   - 杜绝向用户做“我是基于XX模型生成的，这可能存在误差...”之类的大段道歉与辩解。
2. **底层 YAML 微创手术 (For `timeline.yaml`)**
   - 运用 `view_file` 定案错误块。
   - 使用文件编辑工具（`replace` 系列）直接斩断原 `event` 或 `summary`，用正确的事件块替换回去，抹除系统修补痕迹。
3. **生平碎片物理剔除 (For `raw_notes/*.md`)**
   - 若用户表示该段落存在“虚空造物”或“加戏情绪”，将其彻底抹除和脱水至客观真相，覆写整文件。
4. **合成章节迭代重映 (For `chapters/*.md`)**
   - 若成书的传记受到指正，记住受矫正的新认知基线，调用 `synthesis.md` 进行全新合成覆盖。文中不能掺杂类似“此章节经作者建议有所修正”等任何破壁词汇。
