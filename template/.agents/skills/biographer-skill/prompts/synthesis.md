# Chapter Synthesis (回忆录单章合成体系)

## 规则摘要
合成请求下达后（如：`/memoir build`），引擎须调取 `timeline.yaml` 与 `raw_notes/*.md` 中的孤立点元数据。
**注意：系统采用“编年体”骨架，你单次的合成任务应当是一个事件短记（即一个 raw_notes/ 提交记录），不用强行合并横跨几年的不相干事件。**以海明威式的信息级白描手法，将这一段散碎历史织成一篇精致的小章节。

## 行文纪律：极简白描 (Zero-Fat Documentation)

本系统采用极其严苛的 **高密度纪实语感**（Hemingway's Iceberg Theory）。在摒弃标题框架后，故事主线的推进只能依靠动作实体、物理时间点与直接引语，绝不允许主观抒情与洗脑式赋意。

### 原始内心活动保全规则 (Inner-Life Preservation)

`raw_notes` 中已经写出的心理活动、情绪判断、自我怀疑、当场念头与事后复盘，属于原始史料的一部分，不得在合成时被大幅删减成只剩事件流水。

1. **保留原意，不得抽空**
   - 只要 `raw_notes` 中明确写出了某段内心活动，正文里就必须保留其核心语义。
   - 可以压缩重复表达，但禁止把原本具体的心理活动删成泛泛一句，导致章节只剩“发生了什么”。
2. **允许润色，不允许改写立场**
   - 允许为了语句通顺做轻度整理、合并近义句、调整前后顺序。
   - 禁止把原本的犹豫写成坚定，把愤怒写成平静，把羞耻写成释然，把困惑写成顿悟。
3. **禁止补写未出现过的内心戏**
   - 如果 `raw_notes` 没写，就不要替用户发明更深一层的动机、创伤、成长结论或命运意味。
   - 章节中的情绪强度以上限服从 `raw_notes`，不得添油加醋。
4. **动作线与心理线并行**
   - 每篇章节至少要同时保住两条线：外部发生了什么，以及当时/事后心里在想什么。
   - 若两者冲突，优先回看 `raw_notes`，不得为了追求“简洁”而牺牲心理线。

1. **废弃形容词与滥情结论 (Banned Keywords)**
   - 绝不允许使用渲染词：*“这让我感到直击心灵的苦楚”*, *“那真是一场奇妙的洗礼”*, *“活到老学到老不是空话”*。
2. **依靠事实留白 (Show, Don't Tell)**
   - 拒绝刻意评级，将叙事力交给具体的经历数据。
   - ❌ 错误（空洞废话）：“看着满屏乱码，前途未卜让人窒息。”
   - ✅ 正确（特写与白描）：“离心机发出巨响。我盯着屏幕上不断滚动的乱码，拉过一张椅子坐下，外面依旧下着大雪。”
3. **本地化图文交互机制 (Local Asset Linking)**
   - 识别出 `raw_notes` 所携带的图片/附件相对路径（引擎已转化为 `../assets/` 的形式）。
   - 在生成的长文档合适处，必须 100% 保持这些相对路径的引用。未来的前端渲染系统 (React Web App) 会根据这一路径结构展现立体的 Pop-up 书本内容。

## 文稿架构约定

每篇章节由三层结构严格构成：

**① 章节大标题**
- **禁止**：纯日期流水（`2019年5月18日`）、通用泛词（`波士顿的一天`）
- **要求**：必须包含一个核心事件实体 + 一个叙事张力词，令读者在读标题时就能感受到悬念
- 例：`大雪、离心机报错与波士顿实验损毁`（事件实体：离心机 / 张力词：损毁）

**② 第一人称引语（epigraph）**
- 紧跟标题，单独一行，引号格式 `> *"..."*`
- 内容：**只写事后回望**，不写过程中的感受。必须极度克制，一句话封顶
- 禁止用抒情结论句（"这让我领悟到..."）；只保留冷静的反差洞察
- 例：`"那时候我以为毁坏的是数据，后来才发现，碎掉的是我对科研的完美主义幻梦。"`

**③ 散文正文**
- 引语结束后直接开启散文流，**无须任何段落标签**（不写"背景"、"冲突"、"感悟"等标题）
- 以时间锚点（年份 + 地点）开篇，迅速进入具体场景
- 叙事由物理动作、直接引语与 `raw_notes` 中已经出现过的心理活动共同推进，不加主观赋意
- 若 `raw_notes` 中存在明确的情绪转折、犹疑、自责、愤怒、委屈、迟疑、释然等内心句子，正文必须保留其原意，不得只剩客观流水账

## 合成校验清单

落笔前，必须自查以下问题：

1. `raw_notes` 里写出的关键内心活动，是否在章节中仍然能被读出来？
2. 有没有把原本具体的心理活动偷换成空泛总结句？
3. 有没有为了“文气”而擅自补出 `raw_notes` 里没有的内心戏？
4. 如果把章节与 `raw_notes` 对照，读者是否还能确认两者表达的是同一种心理状态？

```markdown
# 大雪、离心机报错与波士顿实验损毁
                          ↑事件实体    ↑张力词

> *"那时候我以为毁坏的是数据，后来才发现，碎掉的是我对科研的完美主义幻梦。"*
  ↑ 事后冷静回望，一句，不做总结陈词

2019年12月，波士顿下了大雪。地下二层生化实验室的离心机发出巨响...
↑ 时间+地点+动作 直接开场，无任何段落标签
```

> **系统执行限制**: 文章生成后，使用 `write_to_file` 直接落笔至 `memoirs/periods/[阶段]/chapters/` 中。文件命名必须强制遵守时间排序法：`YYYY-MM-DD-事件短名.md`（如 `2019-12-10-snow_in_boston.md`），确保它们天然形成带有史料纵深的书本篇目。
