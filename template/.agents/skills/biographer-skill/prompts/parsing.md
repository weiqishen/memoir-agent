# Fragment Parsing (碎片影像入库规范)

## 任务目标
当用户输入未经整理的碎片时（文字吐槽、旧照、原稿），通过强制逻辑剥除冗余，将核心要素提取与归属，并使用底层脚本安全落盘。

## 提取指令 (Structured Extraction)

1. **时间打底 (Anchor Date)**
   - 寻找材料真实支持的时间粒度：精确日（`2019-05-18`）、月份（`2019-05`）、年份（`2019`）、季度（`2019-Q3` / `2019年第三季度`）或约略年份（`约2019年`）。
   - 不得为了满足格式而编造具体日期；如果来源只支持年份或季度，就按粗粒度写入 `--date`。
   - 裸月份（如 `9月`）必须先结合现有 timeline 推断明确年份；仍无法确定时，立即暂停执行并追问。
   - 对非精确日期的事件，优先使用稳定 `--file-slug`，避免后续标题或时间修订影响图谱索引。
2. **场景复刻 (Visual Scan)**
   - 提取照片中的信息变量：微气候特征、特定物品、特殊仪器状态。不加主观评价。
3. **三维拆解 (Triple-Factor Isolation)**
   - **Context (客观背景)**：纯纪实的事件起因、人物交织及其物理环境。
   - **Emotion (真实情绪)**：绝对客观地保留核心态势（不论是狂喜、绝望，还是极致的无聊与宁静），禁止强加"挣扎或冲突"。
   - **Reflection (事后感语)**：材料中是否夹带着跳脱出事件轴的复盘？有则取之。
4. **本地化存档图库 (Local Asset Archival)**
   - 提取多媒体素材并保留原始绝对路径（例如 `E:/Photos/2005.jpg`）在其原本的 Markdown `<img src="">` 或 `![]()` 格式中。
   - **机制要求**：底层 I/O 引擎 (`timeline_manager.py`) 将负责正则匹配并物理拷贝这些绝对路径资产到本地对应时期的 `assets/` 目录中，以生成独立的、可移植的立体书效果。你只管精准提取带绝对路径的 Markdown 即可，不得自作主张修改引用路径，必须触发底层引擎的拷贝机制。
5. **属地深度推断与防重机制 (Deep Inference & De-duplication)**
   - **机制要求**：在判定阶段分类名（`--period`）前，你**必须**执行一轮"深度推断提取"：
     1. 调用 `list_dir` 探查 `memoirs/periods/` 获取当前所有已建立的阶段（如 `US_PhD`）。
     2. 如果新到碎片的时间（如 2020年）或环境有重叠嫌疑，你必须调用工具（如 `view_file` 或 `grep_search`）阅读对应目录下的 `timeline.yaml`，通过比对现有事件的时间轴跨度和背景，进行逻辑归类。
   - 优先将新碎片通过这种基于上下文的深度比对，安全吸附到在时间跨度和逻辑上已存在的生命周期中。绝对杜绝分裂出别名（如明明已有跨越 2018-2022 的 `US_PhD`，却凭空生造出一个独立的 `2020_Pandemic` 或 `Boston_Years`）。
   - 只有当经过实证探查，极度确信完全属于全新生命大篇章时，才可生造全局新目录（如 `--period "Postdoc"`）。
6. **两类实体字段提取 (Entity Field Extraction)**

   raw note frontmatter 中的两个独立字段各有明确职责：

   | 字段 | 收录内容 | 用途 |
   |------|---------|------|
   | `--people` | 所有出现的真实人物（如：`老王,导师张教授`） | 人物视图 + 知识图谱节点 |
   | `--places` | 地点（**必须使用完整限定名**，见下方规范） | 地点视图 + 知识图谱节点 |

   **地点完整限定名 (FQN) 规范**：
   - ✅ 顶级地点直接写：`虹桥火车站`
   - ✅ 子地点必须带父前缀：`虹桥火车站·二楼候车厅`（用中点 `·` 分隔）
   - ❌ 禁止写裸骨子地点名：`二楼候车厅`（不同火车站都有，歧义）
   - 如果某个子地点在 `memoirs/entities.yaml` 中已定义了 FQN，直接用；否则自行构造 `父地点·子地点` 格式
   - 人物和地点 alias 写入 `memoirs/entities.yaml` 的 `aliases` 字段；同一实体的中文名、英文名、简称、外号都应登记为 alias。
   - 已登记的父地点 alias 会自动和子地点 alias 组合：例如 `UF` + `commuter lot` 可解析为 `佛罗里达大学·通勤停车场`。
   - 如果只知道裸子地点名且可能歧义（如 `library`、`lobby`、`大厅`），不要直接写入 `--places`；必须补足父地点或先追问。
   - 编译器会把歧义 alias 写入 `memoirs/.entity_resolution_report.json`，但入库阶段仍应尽量输出明确 FQN，减少后续人工清理。

## 系统落地引擎

要素提纯后，生成带参指令供底层执行器完成数据库与树结构的创建，自身不得代写文本：

```bash
# 示例执行脚本（period 文件夹会自动在 memoirs/periods/ 下创建）
python .agents/skills/biographer-skill/tools/timeline_manager.py \
    --action append \
    --period "Undergrad" \
    --file-slug "2010_09_college_arrival" \
    --date "2010-09" \
    --event "开学报到遭遇台风" \
    --people "妈妈" \
    --places "上海虹桥火车站,上海虹桥火车站·大厅" \
    --summary "人生第一次离家，遇上台风天滞留了8个小时。" \
    --context "拎着三个蛇皮袋在售票窗口下苦等。车站顶棚甚至都发生漏水。" \
    --conflict "狼狈不堪，充满去到陌生城市的彷徨与失落感。" \
    --reflection "当时只想要张买回家的票，完全没意识到那就是独立的起点。" \
    --raw-input "这是我妈给我拍的在火车站门口的一张傻瓜照：![雨中相送](H:/Photos/2010/shanghai.jpg)"
```

`--date` 可使用粗粒度值，例如 `2010`、`2010-09`、`2010-Q3`、`2010年第三季度`、`约2010年`。底层编译器会在 manifest 中生成规范化 `time` 元数据；入库时不要把不确定的季度或年份硬写成某一天。
